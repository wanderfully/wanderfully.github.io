正则与单词相记忆

符号 描述 记忆方式
. 任意单个字符，换行符除外 句号
\w 任意一个字母数字或下划线，0-9a-zA-Z word
\W 查找非单次的字符等价于[^A-Za-z0-9] not word
\d 匹配一个数字字符，等价于[0-9] digit
\D 匹配不是数字字符的，等价于[^0-9] not digit
\s 匹配任何空白字符，等价于[\f\n\r\t\v] space
\S 匹配任何非空白字符，等价于[^\f\n\r\t\v] not space
\b 匹配一个单词边界，比如’er\b’可以匹配”never”中的”er”,但是不能匹配”verb”中的”er” boundary
\B 匹配一个非单词边界,’er\B’能匹配’verb’中的’er’,但不能匹配’never’中的’er’ not boundary
\0 匹配非单词边界,’er\查找NUL字符
\n 换行符 new line
\f 换页符 form feed
\r 回车符 return
\t 制表符 tab
\v 垂直制表符 vertical tab
\xxx 以八进制数xxx规定的字符
\xdd 查找以16进制数dd规定的字符
\uxxxx 以16进制数的xxxx规定的Unicode字符

量词

用于限定子模式趁现在正则表达式中的次数

符号 描述
+ 匹配一次或多次，相当于{1,}
* 匹配零次或多次 ，相当于{0,}
? 匹配零次或一次 ，相当于{0,1}
{n} 匹配n次
{n,m} 匹配至少n个，最多m个某某的字符串
{n} 匹配至少n个某字符串

位置符号

符号 描述
$ 结束符号，以什么结尾
^ 起始符号，以什么开始
?= 肯定正向环视,例：?=n,匹配其后紧接指定的n字符串
?! 否定正向环视,例如：?!n,匹配其后没有紧接指定的n字符串
?: 表示不匹配

正则表达式中的 ?= 和 ?: 和 ?! 的理解

零宽正向断言：

exp1(?=exp2) 括号内表示某个位置右边必须和=右边匹配上

负向断言

exp1(?!exp2) 括号内表示某个位置右边不和!后的字符匹配

后顾：

(?<=exp2>)exp1 查找exp2后面的exp1

负后顾：

(?!<=exp2)exp1 查找前面不是exp2的exp1

例子：

“中国人”.replace(/(?<=中国)人/, “rr”) // 匹配中国人中的人，将其替换为rr，结果为 中国rr

“法国人”.replace(/(?<=中国)人/, “rr”) // 结果为 法国人，因为人前面不是中国，所以无法匹配到

要理解?:则需要理解捕获分组和非捕获分组的概念：

()表示捕获分组，()会把每个分组里的匹配的值保存起来，使用$n(n是一个数字，表示第n个捕获组的内容)

(?:)表示非捕获分组，和捕获分组唯一的区别在于，非捕获分组匹配的值不会保存起来

举例：

// 数字格式化 1,123,000

“1234567890”.replace(/\B(?=(?:\d{3})+(?!\d))/g,”,”) // 结果：1,234,567,890，匹配的是后面是3*n个数字的非单词边界(\B)

match： stringObject.match(regxp) match返回的一个数组，第一个元素是整体匹配结果，然后是各个分组（括号里）匹配的内容，然后是匹配下标，最后是输入的文本。（注意：如果正则是否有修饰符g，match返回的数组格式是不一样的）

例：

console.log(‘30CACDVB0040’.match(/(\d{1,2})/g)); // [“30”,”00”,”40”]

exec： regex.exec(sring)

replace： str.repace(regxp,”替换成何值”) /g 全局替换（global） /i 忽略大小写（ignore忽略）

例：

var str =’学 习 正 则’;
console.log(str.replace(/\s+/g,’?’)); // 学?习?正?则

test： 验证字符窜是否符合该正则表达式

例：

var str =’学 习 正 则’;
console.log(/^\d$/g.test(‘123789’)); // true
console.log(/^\d$/g.test(‘12378b9’)); // false

反向引用 对分组 可以进行引用 \1：表示引用第一个分组 \2：表示引用第二个 依次类推
